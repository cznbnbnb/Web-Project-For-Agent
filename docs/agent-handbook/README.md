## 配置 AI

### 支持的 Provider

| Provider | 描述 | 获取 API Key |
|----------|------|-------------|
| **Anthropic** | Claude 系列模型 | https://console.anthropic.com |
| **OpenAI** | GPT-4o 等官方模型 | https://platform.openai.com |
| **自定义** | DeepSeek、Moonshot 等兼容接口 | 各自官网 |

### 配置步骤

1. 点击顶部导航栏的 **AI 设置**
2. 选择 AI 服务商
3. 填入 API Key
4. 选择模型
5. 保存配置

### 推荐模型配置

```yaml
# 高质量分析
Provider: Anthropic
Model: claude-sonnet-4-20250514

# 性价比
Provider: OpenAI
Model: gpt-4o-mini

# 国内访问
Provider: 自定义
Base URL: https://api.deepseek.com
Model: deepseek-chat
```

## 快捷命令

### 项目分析

| 命令 | 描述 |
|------|------|
| `分析项目` | 分析当前项目计划，识别关键路径和风险 |
| `分析风险` | 识别潜在的项目风险 |
| `关键路径` | 显示关键路径任务列表 |
| `生成报告` | 生成项目状态报告 |

### 任务管理

| 命令 | 描述 |
|------|------|
| `创建任务：[名称]，工期 [X] 天` | 创建新任务 |
| `添加阶段：[名称]` | 添加项目阶段 |
| `添加里程碑：[名称]` | 添加里程碑 |

### 排程管理

| 命令 | 描述 |
|------|------|
| `重新排程` | 重新计算项目排程 |
| `重排计划` | 同上 |
| `计算关键路径` | 重新计算并标记关键路径 |

## Claude Code 集成

### 使用场景

本应用的核心卖点是与 Claude Code CLI 的深度集成，支持通过命令行直接操作项目计划。

### 基本用法

```bash
# 在项目目录下使用 Claude Code
claude "查看当前项目计划"

# 创建任务
claude "创建任务 'UI 设计'，工期 3 天，负责人李四"

# 分析项目
claude "分析项目风险，给出优化建议"

# 调整计划
claude "将所有设计任务延后 2 天"
```

### 快捷脚本

在项目根目录创建 `.claude/commands/project.md`:

```markdown
# 项目计划管理命令

## /project-status
查看当前项目状态概览

## /project-add <任务描述>
添加新任务到项目计划

## /project-analyze
分析项目计划，输出关键路径和风险

## /project-reschedule
重新排程项目计划
```

### 自动化示例

创建 `.claude/hooks/config.json`:

```json
{
  "hooks": {
    "before-commit": "检查项目计划是否已更新"
  }
}
```

## 常见问题

### Q: AI 响应不准确怎么办？
A: 尝试提供更多上下文信息，或切换到更强大的模型如 Claude Sonnet。

### Q: 如何确保数据安全？
A: 所有数据存储在本地 IndexedDB，AI 调用仅发送必要的任务数据结构。

### Q: 支持哪些 Excel 格式？
A: 支持 .xlsx 格式导入导出，兼容 Microsoft Project 导出格式。

### Q: 节假日如何配置？
A: 在设置中添加节假日日期，排程时会自动跳过。
